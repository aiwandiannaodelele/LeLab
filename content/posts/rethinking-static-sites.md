> 速度不是一种特性，而是一种态度。

把博客做成静态生成，意味着每一次发布都是一次"凝固"。内容在构建时就被渲染成 HTML，服务器不再需要为每一次请求去查询数据库、拼装模板。读者拿到的是已经被精心排版过的页面，毫秒之间就能开始阅读。

## 为什么是静态？

过去几年，我们从 LAMP 一路走到 JAM，再到各种全栈框架。但博客这种以内容为主体、更新频率不高的形态，其实最适合静态。原因有三：

- **性能**：没有运行时的查询，CDN 边缘节点直接返回。
- **稳定**：没有数据库就没有数据丢失，没有后端就没有后端宕机。
- **自由**：构建产物是一堆 HTML/CSS/JS，可以放在任何地方。

## 不妥协的部分

静态不等于简陋。现代构建链可以在构建时完成相当复杂的逻辑：

```ts
// 构建时遍历所有 markdown 文章
for (const file of posts) {
  const { data, content } = matter(file)
  const html = await render(content)
  yield { ...data, html }
}
```

按标签聚合、生成归档页、计算阅读时间、提取目录——这些都是构建时能完成的事，运行时零成本。

## 就这样

这会是一个长期在更新的角落。文字会被慢慢填进来，而框架背后的复杂性，希望能尽量留在这层壳之外。
